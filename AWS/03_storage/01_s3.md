# 01. What is Amazon S3

Amazon Simple Storage Service (S3) is an **object storage service** that allows you to store and retrieve **any amount of data** from anywhere on the web.

- **Object storage** → Data is stored as objects (file + metadata + unique ID)
- **Highly durable** → 99.999999999% (11 9’s) durability
- **Scalable** → Automatically scales for any amount of data
- **Serverless** → No need to manage servers

---

# 02. How S3 Works

1. Create a **Bucket** (container for objects)
2. Upload **Objects** (files, images, videos, documents)
3. Configure **Access permissions** and **Storage classes**
4. Retrieve or manage objects via **AWS Console, CLI, or SDK**

**Key points:**

- S3 is **region-based**, data can be replicated across regions
- Supports integration with almost all AWS services

---

# 03. Key Components

### 3.1 Bucket
- Logical container for objects
- Globally unique name
- Can enable versioning, logging, and encryption

### 3.2 Object
- File + metadata + unique ID (key)
- Max object size: 5 TB

### 3.3 Key
- Unique identifier for each object in a bucket

### 3.4 Versioning
- Keep multiple versions of the same object
- Helps recover from accidental deletions or overwrites

### 3.5 Access Control
- Bucket Policies
- IAM Policies
- Access Control Lists (ACLs)

# 05. Security Features

- **Encryption**
  - SSE-S3, SSE-KMS, SSE-C
  - Client-side encryption
- **Access control**
  - IAM Policies
  - Bucket Policies
  - ACLs
- **MFA Delete** → Extra security for object deletions
- **Logging & Monitoring**
  - CloudTrail integration
  - S3 Access Logs

---

# 06. Pricing Model

- Storage cost per GB per month (varies by storage class)
- Requests & data retrieval costs (GET, PUT, LIST, COPY)
- Data transfer out of S3 (charged separately)

> Example: S3 Standard ~ $0.023/GB/month, Standard-IA ~$0.0125/GB/month

# 07. Example Services That Use S3

- **Glacier** → Archive S3 objects automatically

# 09. Difference: S3 vs EBS vs EFS

| Feature       | S3                     | EBS                 | EFS                     |
|---------------|-----------------------|-------------------|------------------------|
| Storage Type  | Object                | Block              | File                   |
| Access        | Internet / SDK        | EC2 only           | EC2 + Lambda           |
| Scalability   | Virtually unlimited   | Limited by volume  | Virtually unlimited    |
| Durability    | 11 9's               | 99.999%            | 99.999%                |
| Use Case      | Backup, archival      | OS disks, databases| Shared file storage     |

---

# 10. Best Practices

- Use **versioning** to prevent accidental data loss
- Enable **server-side encryption** for sensitive data
- Apply **least privilege IAM roles**
- Use **lifecycle policies** to optimize cost
- Enable **logging and monitoring** for compliance

---

# 11. Interview / Exam Important Notes

- S3 is **object storage**, not block storage
- Max object size: 5 TB, bucket can store unlimited objects
- Supports **event notifications** → Lambda, SQS, SNS
- Supports **cross-region replication**
- **Static website hosting** available
