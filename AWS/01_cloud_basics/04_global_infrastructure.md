# AWS Global Infrastructure

## Regions

- Physical geographic locations
- Each region is isolated from others
- Data residency compliance support

---

## Availability Zones (AZ)

- Multiple AZs per region
- Independent power, cooling, networking
- Designed for high availability

---

## Edge Locations

- Used by CloudFront and Route 53
- Reduces latency
- Improves performance

---

## Important Concept

Multi-AZ = High availability  
Multi-Region = Disaster recovery strategy
